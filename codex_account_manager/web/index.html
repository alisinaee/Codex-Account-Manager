      Guide & Help
    
    

      
Use this panel to add accounts, monitor live usage, import or export saved profiles, tune native notification and auto-switch behavior, review release notes, and update the app from one place.

      

        

          
Quick Start

          
    
            
  • Use Add Account to open the combined login dialog, enter a profile name, then choose Device Login or Normal Login.
    • 
            
  • Use Switch on any row to activate a saved profile in Codex; the active account gets the green status dot and stays pinned first.
    • 
            
  • Use Refresh for a full immediate sync after switching, login, config changes, or recovering from stale data.
    • 
            
  • Current client support targets the Codex CLI and the Codex VS Code extension.
    • 
            
  • On some OS/client combinations, the CLI or VS Code extension may still need a manual reload or restart after switching before the new account is picked up.
    • 
          

        

        

          
Header & Panel Controls

          
    
            
  • Header shows the current app version, and when a newer release exists it adds a green Update available badge plus an Update button.
    • 
            
  • Header controls cycle Theme, toggle Debug mode, and show or hide the settings panels.
    • 
            
  • Current Account Auto Refresh polls only the active account in seconds and is meant for lightweight continuous monitoring.
    • 
            
  • Auto Refresh All runs a slower background sweep in minutes and refreshes saved accounts one by one instead of doing one expensive batch on every tick.
    • 
            
  • Restart restarts the local UI service, and Kill All force-stops managed background account processes.
    • 
          

        

        

          
Profiles Import / Export

          
    
            
  • Import opens an archive picker, analyzes the migration file, and lets you review per-profile actions before anything is applied.
    • 
            
  • Export opens a profile-selection dialog with bulk select actions and an optional custom archive filename.
    • 
            
  • Migration archives include sensitive auth data. Keep them private and only import files from sources you trust.
    • 
          

        

        

          
Native Notifications

          
    
            
  • Use the Notification card toggle to turn native usage notifications on or off.
    • 
            
  • Test Notification sends a real OS notification using the current active profile usage.
    • 
            
  • Set warning thresholds with 5H notify % and Weekly notify %.
    • 
            
  • When warning thresholds are breached, notifications use the same native OS path as the test button.
    • 
            
  • Before a delayed auto-switch executes, the app also sends a native heads-up about 30 seconds before the switch time.
    • 
          

        

        

          
Auto-Switch Rules

          
    
            
  • Execution: toggle Enabled, set Delay (sec), run Run Switch/Rapid Test/Stop Tests/Test Auto Switch.
    • 
            
  • Selection Policy: choose ranking mode and set switch thresholds.
    • 
            
  • Switch Chain Preview shows current order; Edit supports manual reorder.
    • 
            
  • Auto Arrange recalculates chain ordering from current usage and keeps the active account locked at the front when editing the chain manually.
    • 
          

        

        

          
Accounts Area

          
    
            
  • Toolbar actions: Add Account, Remove All, and Columns.
    • 
            
  • Table supports sorting by headers and optional column visibility control.
    • 
            
  • Per-row actions: Switch, row menu (Rename/Remove), and Auto eligibility toggle.
    • 
            
  • Plan and Paid fields are available as optional columns (hidden by default).
    • 
            
  • Mobile cards open full detail modal and include switch + row-actions shortcuts.
    • 
            
  • Tooltips are available across controls, headers, and row actions to explain what each setting or button does.
    • 
            
  • Current-account usage is read from the live active auth session, and healthy current sessions sync back only when identity matches to avoid stale-token drift without cross-profile overwrite risk.
    • 
            
  • If a row shows auth expired, the saved profile auth snapshot no longer works for usage API calls and that account needs a fresh healthy session.
    • 
          

        

        

          
Debug & Advanced

          
    
            
  • In debug mode, System.Out shows UI/API actions, warnings, events, and errors.
    • 
            
  • Export Debug Logs downloads a JSON snapshot for troubleshooting.
    • 
            
  • Advanced Actions exposes wrapped `codex-auth` operations for status/login/switch/import/config/daemon/clean/raw auth when deeper inspection is needed.
    •
